## 0. 前提

### SourceRef の形状（`src/types/domain.ts`）

```ts
export type SourceRef = {
  id: string;      // 一意キー（例: "pmda-moclobemide-if"）
  title: string;   // 出典タイトル
  url?: string;    // 参照 URL（任意）
  note?: string;   // 補足（版・参照箇所など、任意）
};
```

### 伝播経路（Task5A で配線済み）

- `interactionRules[].sources`（未整備）→ `evaluateInteractions` が発火ルールの sources を連結
  → `calculateRisk` / `calculateCombinedRisk` の返り値 → `RiskResult.sources` / `CombinedRiskResult.sources`
  → UI（`/about` の出典一覧、ScorePanel 周辺）で表示可能。
- **配線は完成済み。欠けているのは各ルールの `sources` データのみ。**

## 2. ルールが主張している内容（要約）と必要出典カテゴリ

各ルールの `description` が主張する臨床的リスクを要約し、それを裏づけるのに適した
出典カテゴリを提案する。**具体の文献 ID・URL はここでは確定しない**（Task5C で実在確認のうえ付与）。
候補として挙げる文献名は「探索の当たり」を示すもので、いずれも **要一次確認**。

### Rule 1: `maoi_plus_serotonergic`（high, +10）

- **主張**: RIMA（モクロベミド）とセロトニン作動性物質の併用はセロトニン症候群リスク。
- **必要出典カテゴリ（優先順）**:
  1. **添付文書 / インタビューフォーム** — モクロベミド、および併用側（トラマドール配合剤）の
     「併用禁忌／併用注意」記載。
  2. **PMDA** — 医薬品リスク管理・安全性情報（セロトニン症候群の注意喚起）。
  3. **公的ガイドライン / 標準教科書** — セロトニン症候群の診断・機序。
  4. **査読論文（レビュー）** — 例: セロトニン症候群の総説（要一次確認）。
- **最低必要件数の目安**: 2（添付文書系1 + 機序/総説1）。high 重大度のため厚めに。

### Rule 2: `depressant_stacking`（caution, +6）

- **主張**: 中枢抑制作用を持つ物質が2種類以上で、過鎮静・呼吸抑制リスク上昇の可能性。
- **必要出典カテゴリ**:
  1. **添付文書** — 対象物質（プレガバリン、エスゾピクロン、トラマドール配合剤 等）の
     「中枢神経抑制薬との併用注意」記載。
  2. **PMDA / 行政の安全性通知** — 併用による鎮静・呼吸抑制の注意喚起。
  3. **査読論文 / ガイドライン** — CNS 抑制薬の相加作用に関する総説。
- **最低必要件数の目安**: 1〜2（添付文書系1 は必須）。

### Rule 3: `opioid_plus_sedative_hypnotic`（high, +8）

- **主張**: オピオイド様作用と睡眠薬様作用の併用は呼吸抑制の相乗リスク。
- **必要出典カテゴリ**:
  1. **規制当局の重大警告** — 例: FDA のオピオイド × ベンゾジアゼピン/CNS 抑制薬の
     枠組み警告（boxed warning）相当（要一次確認）。
  2. **添付文書** — オピオイド様（トラマドール配合剤）・睡眠薬様（エスゾピクロン）の併用注意。
  3. **PMDA** — 併用による呼吸抑制の注意喚起。
  4. **査読論文** — 併用と過量・呼吸抑制の疫学（要一次確認）。
- **最低必要件数の目安**: 2（当局警告/添付文書1 + 機序または疫学1）。high 重大度のため厚めに。

### Rule 4: `seizure_threshold_with_stimulant`（caution, +4）

- **主張**: けいれん閾値を下げる物質と中枢刺激作用の併用はけいれん誘発リスク。
- **必要出典カテゴリ**:
  1. **添付文書** — トラマドール（けいれんリスク）、および刺激系（メチルフェニデート）の記載。
  2. **査読論文 / 症例報告** — けいれん閾値低下と刺激薬併用の相互作用。
  3. **公的ガイドライン** — けいれんリスク管理。
- **最低必要件数の目安**: 1〜2（添付文書系1 は必須）。

## 3. 出典カテゴリの定義（この4カテゴリに揃える）

Task5C で `SourceRef.note` や `id` プレフィックスに使う分類を先に固定する。

| カテゴリ | 内容 | id プレフィックス案 | 信頼度の位置づけ |
|---|---|---|---|
| 添付文書 / IF | 各製剤の添付文書・インタビューフォーム | `pi-` / `if-` | 一次（規制承認情報） |
| PMDA / 行政 | PMDA 安全性情報、行政の通知・DIメモ | `pmda-` | 一次（当局） |
| 公的ガイドライン | 学会・公的機関の診療/安全ガイドライン | `guideline-` | 二次（合意形成） |
| 査読論文 | 査読付き総説・原著・症例報告 | `paper-` | 二次（個別エビデンス） |

> 規制当局警告（FDA boxed warning 等の海外当局情報）は運用上「PMDA / 行政」カテゴリに含めるか、
> 別途 `regulator-` を新設するかを Task5C 着手時に決める（本計画では未確定）。
