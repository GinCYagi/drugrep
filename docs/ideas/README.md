# docs/ideas/

将来機能の**調査・設計メモ**置き場。

## このディレクトリの位置づけ

- ここにあるものは **実装済みではない**。コードには存在しない構想を含む。
- いずれも **MVP対象外**。現時点で着手する予定はない。
- ルートの `README.md` や アプリ内 `/about` には **未反映** の構想を含む
  （＝プロダクトの公開仕様ではない）。
- 「調べたが今はやらない」という **実装判断前の設計資産** として保存し、
  再調査の重複を防ぐことが目的。

## 注意: これらのメモは陳腐化する

コードの事実（物質一覧・`substanceId`・エイリアス等）を引用する場合、
それは執筆時点のスナップショットに過ぎない。**唯一の正は `src/lib/substances.ts`**
（およびロジックは `src/lib/`）であり、メモとコードが食い違ったらコードが正しい。

そのため各メモには「最終照合日」を必ず持たせ、引用箇所には
「YYYY-MM-DD時点」であることが分かる形で書く。

## メモの書き方

- 1メモ = 1ファイル（Markdown）。
- 冒頭に必ずメタブロックを置く:

  ```
  > ステータス: <ステータス語彙のいずれか>
  > 作成日: YYYY-MM-DD
  > 最終照合日: YYYY-MM-DD（コードと突き合わせた日）
  ```

- 未確定事項は本文中に **要確認** マーカーを付け、末尾のチェックリストに集約する。

### ステータス語彙（この一覧に統一）

| ステータス | 意味 |
|---|---|
| アイデア段階 | 調査前／着想のみ |
| 現時点では実装しない（Post-MVP候補） | 調査済み・MVP対象外・将来やる可能性あり |
| 廃案 | 検討の結果やらないと判断 |

## ライフサイクル

実装が決まったメモは、`docs/` 直下や issue に昇格させ、
このディレクトリからは削除またはアーカイブ扱いにする
（`docs/ideas/` には「まだやらないもの」だけを残す）。
