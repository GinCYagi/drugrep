### 補足（層の設計・レビュー時の前提）

- スコア非表示（S4/S6/S7/S9/N2）は **UI 層の責務**。ロジック層（`calculateRisk`/`calculateCombinedRisk`）は範囲外でも数値を返す（golden 固定済み）。
- 範囲外の判定は UI が独自に dose 比較せず、ロジック層 `outOfModelRangeReason`（構造化コード `ScoreSuppressionReason`）に一本化。
- S9 の一括抑制は裁定 D（2026-07-07・Gin 承認）: 構成要素が1つでも範囲外なら合算値を数値として信頼しない。
- **境界の期待値（要注視）**: `dose <= veryHighMax`（=120）は段階2＝**範囲内でスコア表示**、`dose > veryHighMax`（=121）で**適用範囲外**。`doseStage`/`getDoseMultiplier` の境界（`<=`）と一致。
- **119救急導線は全 out-of-range 状態で第一級の確認対象**: S4/S7/S9/N2 で、スコア非表示でも「緊急時は119へ」（上部短文＋フッター）が**存在し可読**であること。スコア抑制と救急導線の非表示を混同させない。
- **S11/S13 は機能確認寄り**（〔機能確認寄り〕表記）。L5 運用を分断しないため本表内に残すが、解釈検証というより表示機能の確認である点に留意する。
